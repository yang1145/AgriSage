# 桂收 · 甘蔗专用版 — 技术栈说明文档

## 一、技术架构总览

| 层级 | 技术选型 | 定位 |
|------|---------|------|
| 后端框架 | Flask 3.0 | 轻量级 Python Web 框架 |
| ORM | Flask-SQLAlchemy 3.1 | 数据库抽象层 |
| 数据库 | SQLite（默认）/ MySQL / PostgreSQL | 关系型数据库 |
| 前端框架 | Vue 3.4 | 渐进式 JavaScript 框架 |

## 二、后端技术栈详解

### 2.1 Flask 3.0 — 轻量级 Web 框架

**技术优势：**

| 优势维度 | 说明 |
|---------|------|
| **轻量简洁** | 核心代码精简，入门门槛低，开发效率高；无默认数据库、模板引擎绑定，按需扩展 |
| **灵活扩展** | 拥有完善的扩展生态（Flask-SQLAlchemy、Flask-CORS、Flask-JWT 等），可根据项目需求自由组合 |
| **RESTful 友好** | 原生支持 REST API 开发，通过装饰器轻松定义路由与视图函数 |
| **异步支持** | Flask 3.0 内置异步支持，可处理高并发请求 |
| **调试方便** | 内置开发服务器，支持热重载，修改代码即时生效 |

**在项目中的应用：**
- 作为 REST API 服务端，处理前端请求
- 通过 Blueprint 模块化组织 API 路由
- 集成 waitress 作为生产环境 WSGI 服务器

---

### 2.2 Flask-SQLAlchemy 3.1 — ORM 数据库抽象

**技术优势：**

| 优势维度 | 说明 |
|---------|------|
| **面向对象** | 数据库表映射为 Python 类，操作数据库如同操作对象，无需编写原生 SQL |
| **防注入** | 内置 SQL 参数转义，有效防止 SQL 注入攻击 |
| **数据库无关** | 同一套代码可切换 SQLite、MySQL、PostgreSQL 等多种数据库 |
| **迁移支持** | 支持数据库迁移脚本，保障版本升级时的数据兼容性 |
| **关系映射** | 原生支持一对一、一对多、多对多关系建模 |

**在项目中的应用：**
- 定义 User、Plot、PlantingCycle、FarmingRecord 等数据模型
- 处理所有数据库 CRUD 操作
- 实现用户权限与数据隔离逻辑

---

### 2.3 Flask-CORS 4.0 — 跨域资源共享

**技术优势：**

| 优势维度 | 说明 |
|---------|------|
| **配置简单** | 一行代码开启跨域支持 |
| **细粒度控制** | 支持指定允许的域名、HTTP 方法、请求头 |
| **安全可控** | 默认仅允许 GET/HEAD/POST 方法，支持 CSRF Token 验证 |

**在项目中的应用：**
- 解决前端开发服务器（localhost:3000）与后端服务（localhost:5000）的跨域问题
- 生产环境下配合 Flask 静态文件托管，实现单点部署

---

### 2.4 PyJWT 2.8 — JSON Web Token 认证

**技术优势：**

| 优势维度 | 说明 |
|---------|------|
| **无状态认证** | Token 自包含用户信息，服务端无需存储 Session |
| **跨域友好** | Token 通过请求头传递，适用于前后端分离架构 |
| **过期控制** | 支持设置 Token 有效期，可实现自动登出 |
| **签名验证** | HS256/RS256 签名算法确保 Token 不可伪造 |

**在项目中的应用：**
- 用户登录后生成 JWT Token 返回前端
- 前端请求携带 Token，后端验证 Token 有效性
- 实现 30 天自动登录（默认 Token 有效期 720 小时）

### 2.6 waitress 3.0 — 生产级 WSGI 服务器

**技术优势：**

| 优势维度 | 说明 |
|---------|------|
| **跨平台** | 纯 Python 实现，Windows/Linux/macOS 通用 |
| **高性能** | 多线程/多进程模型，支持高并发请求 |
| **稳定可靠** | 生产环境验证，广泛应用于大规模部署 |
| **轻量安装** | 作为 Flask 内置服务器的替代，无需复杂配置 |

---

### 2.7 SQLite — 轻量级嵌入式数据库

**技术优势：**

| 优势维度 | 说明 |
|---------|------|
| **零配置** | 无需安装数据库服务，文件即数据库 |
| **单文件** | 所有数据存储在单一 `.db` 文件，便于备份、迁移 |
| **性能优秀** | 对于中小规模数据（< 100GB），读写性能不逊于 MySQL/PostgreSQL |
| **离线可用** | 完全本地化运行，无需网络连接 |
| **跨平台** | 数据库文件可在 Windows/Linux/macOS 之间自由迁移 |

**在项目中的应用：**
- 默认数据库存储所有业务数据
- 支持一键备份到 U 盘或云同步目录
- 可切换为 MySQL/PostgreSQL 应对大规模数据场景

## 三、前端技术栈详解

## 四、技术架构优势总结

### 4.1 前后端分离架构

| 优势 | 说明 |
|------|------|
| **独立开发** | 前后端可并行开发，通过 API 接口约定协作 |
| **技术灵活** | 前后端可独立选择最适合的技术栈 |
| **专业分工** | 前端专注用户体验，后端专注业务逻辑 |
| **部署解耦** | 可独立部署前后端服务，灵活扩展 |

### 4.2 离线优先设计

| 优势 | 说明 |
|------|------|
| **无网络可用** | SQLite 本地数据库，100% 核心功能离线可用 |
| **数据本地化** | 数据不出村，保障农户数据安全 |
| **离线地图** | 支持预下载地图瓦片，离线地块绘制 |
| **快速响应** | 本地数据读写无网络延迟 |

### 4.3 现代化开发体验

| 优势 | 说明 |
|------|------|
| **Vite 极速 HMR** | 开发修改即时预览，开发效率倍增 |
| **组件化开发** | Vue 3 组合式 API，代码复用性高 |
| **类型安全** | TypeScript 支持（可选），减少运行时错误 |
| **自动化构建** | Vite 自动代码分割、Tree-shaking、懒加载 |

### 4.4 生产级稳定性

| 优势 | 说明 |
|------|------|
| **waitress 服务器** | 生产环境稳定运行，支持高并发 |
| **数据库迁移** | SQLAlchemy 支持数据迁移脚本 |
| **错误处理** | 统一的异常捕获与日志记录 |
| **备份恢复** | 单文件数据库，一键备份/恢复 |

## 五、技术选型对比

### 5.1 后端框架对比

| 维度 | Flask | Django | FastAPI |
|------|-------|--------|---------|
| 轻量程度 | ★★★ | ★★ | ★★★ |
| 学习曲线 | ★★ | ★★★ | ★★ |
| 性能 | ★★ | ★★ | ★★★ |
| 扩展生态 | ★★★ | ★★★ | ★★ |
| 适用场景 | 中小 API / 微服务 | 全功能 Web 应用 | 高性能 API / 异步 |

**选择 Flask 的理由：** 项目定位为本地化部署的轻量级应用，Flask 的灵活性和扩展性完全满足需求，同时保持代码简洁易懂。

### 5.2 前端框架对比

| 维度 | Vue 3 | React | Angular |
|------|-------|-------|---------|
| 学习曲线 | ★★ | ★★ | ★★★ |
| 体积大小 | ★★★ | ★★ | ★ |
| 灵活性 | ★★★ | ★★ | ★ |
| 组件生态 | ★★ | ★★★ | ★★★ |
| TypeScript | ★★★ | ★★ | ★★★ |

**选择 Vue 3 的理由：** 渐进式架构适合项目从简单到复杂的演进，Composition API 提供更好的逻辑复用，Element Plus 组件库与 Vue 3 完美契合。

### 5.3 数据库对比

| 维度 | SQLite | MySQL | PostgreSQL |
|------|--------|-------|------------|
| 部署难度 | ★★★ | ★★ | ★ |
| 性能（中小数据） | ★★★ | ★★ | ★★ |
| 并发支持 | ★ | ★★★ | ★★★ |
| 功能丰富度 | ★★ | ★★★ | ★★★ |
| 离线适用 | ★★★ | - | - |

**选择 SQLite 的理由：** 项目强调本地化部署与离线可用，SQLite 单文件特性完美匹配需求，同时支持平滑升级到 MySQL/PostgreSQL。
